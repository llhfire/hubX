# 线索成本模块设计

## 背景

现有系统是基于 React、Vite、Arco Design 的 CRM 原型，已经包含线索管理、公海线索、我的线索、销售跟进等模块。本设计在现有系统中新增一级菜单“线索成本”，作为推广获客成本管理功能模块。

模块目标是帮助优化师和推广主管记录渠道投放、平台充值、初始线索验证、申诉退款，并按渠道计算单条真实线索成本和综合效果评分。

## 第一版范围

第一版为前端原型展示版：

- 新增一级菜单“线索成本”
- 新增 5 个页面：成本看板、投放日报、充值记录、初始线索池、渠道分析
- 使用本地模拟数据展示业务流程
- 支持弹窗、按钮、状态流转等前端交互演示
- 不接后端接口
- 不做真实登录权限
- 不做真实数据持久化
- 不真正写入现有销售线索模块，只模拟“已分配给销售”状态

## 菜单与路由

在左侧菜单中新增一级菜单“线索成本”，位置放在“线索管理”之后、“客户管理”之前。

菜单结构：

```text
线索成本
├── 成本看板
├── 投放日报
├── 充值记录
├── 初始线索池
└── 渠道分析
```

路由与页面组件：

| 菜单 | 路由 | 页面组件 |
|------|------|----------|
| 成本看板 | `/lead-cost/dashboard` | `LeadCostDashboard` |
| 投放日报 | `/lead-cost/daily` | `LeadCostDaily` |
| 充值记录 | `/lead-cost/recharge` | `LeadCostRecharge` |
| 初始线索池 | `/lead-cost/raw-leads` | `LeadCostRawLeads` |
| 渠道分析 | `/lead-cost/analysis` | `LeadCostAnalysis` |

建议文件结构：


## 页面设计

### 成本看板

用途：让推广主管快速了解整体推广效果和渠道排名。

页面包含：

- 顶部统计卡片：总消耗、有效线索数、平均线索成本、最佳渠道
- 平均线索成本同时展示名义成本和实际成本
- 渠道排名表：渠道、消耗、有效线索、无效线索、有效率、名义成本、实际成本、客资质量、综合评分、排名
- 点击渠道行可跳转或筛选到该渠道的投放日报记录

### 投放日报

用途：优化师每天按渠道记录推广消耗和线索结果。

列表能力：

- 按日期范围、平台、渠道、录入人筛选
- 展示每条投放日报记录
- 支持新增日报

新增日报字段：

- 日期
- 平台
- 渠道/账户
- 消耗金额
- 有效线索数
- 无效线索数
- 录入人

提交后前端计算名义成本和实际成本，并将记录追加到表格状态中。

### 充值记录

用途：记录平台充值和赠送金额，为成本分析提供背景。

列表能力：

- 按日期范围、平台、充值人筛选
- 展示充值记录
- 顶部汇总本月充值总额和赠送总额
- 支持新增充值记录

新增充值字段：

- 充值日期
- 充值平台
- 充值金额
- 赠送金额
- 充值人

### 初始线索池

用途：保存从百度、淘宝、威客、小红书等渠道获取的初始线索，并完成验证、申诉、分配流程。

列表能力：

- 按平台、验证状态、申诉状态、日期范围筛选
- 展示初始线索明细
- 支持新增初始线索

初始线索字段：

- 线索 ID
- 来源平台
- 来源渠道/账户
- 手机号
- 客户需求
- 录入人
- 录入时间
- 验证状态：待验证 / 有效 / 无效
- 验证人
- 验证时间
- 无效原因
- 申诉状态：未申诉 / 申诉中 / 申诉成功（退款）/ 申诉失败
- 退款金额
- 是否分配给销售

操作流程：

1. 新增初始线索后进入“待验证”状态
2. 点击“验证”，选择有效或无效
3. 验证有效后可点击“分配销售”，状态变为已分配
4. 验证无效后可发起申诉
5. 申诉状态可模拟流转为申诉成功或申诉失败
6. 申诉成功时填写退款金额，该金额参与实际成本计算

### 渠道分析

用途：对整体渠道进行横向比较。

页面包含：

- 日期范围和平台筛选
- 消耗金额与有效线索数对比图
- 成本、有效率、线索量、客资质量四维对比图
- 渠道明细表：名义成本、实际成本、有效率、线索量、客资质量、退款影响、综合评分
- 导出按钮只展示 UI，第一版不实现真实导出

## 数据口径

### 名义成本

```text
名义成本 = 消耗金额 ÷ 有效线索数
```

名义成本不考虑充值赠送金额和退款金额，用于直观观察当天投放消耗换来的有效线索成本。

### 实际成本

```text
实际成本 = (消耗金额 - 退款金额) ÷ 有效线索数
```

实际成本扣除申诉成功后的退款金额，用于观察退款后的真实获客成本。

### 充值赠送金额

充值赠送金额不直接摊入单条投放日报成本。第一版在充值记录和渠道分析中展示赠送金额，用于解释平台余额和投放背景。

### 综合评分

```text
综合评分 = 成本得分 × 30% + 有效率得分 × 25% + 线索量得分 × 20% + 客资质量得分 × 25%
```

评分维度：

- 成本得分：有效线索平均成本越低，得分越高
- 有效率得分：有效线索数 ÷ (有效线索数 + 无效线索数)，比例越高得分越高
- 线索量得分：有效线索绝对数量越多，得分越高
- 客资质量得分：高意向等级线索占比越高，得分越高

每项得分按 0-100 归一化，最终加权汇总。

## 与现有 CRM 的关系

- “线索成本”聚焦推广投放、初始线索验证、成本核算和渠道分析
- 现有“线索管理”继续承担销售跟进流程
- 初始线索验证为有效后，第一版只模拟分配销售，不真正写入 `PublicLeads` 或 `MyLeads`
- 后续接后端后，可将有效初始线索转换为现有 CRM 线索记录，并进入销售跟进流程

## 验收标准

1. 左侧菜单出现“线索成本”，包含 5 个二级菜单
2. 5 个路由均可访问，刷新页面后仍可展示
3. 成本看板展示总消耗、有效线索数、平均成本、最佳渠道和渠道综合排名
4. 投放日报可新增记录，并在表格中即时展示
5. 充值记录可新增记录，并更新充值总额和赠送总额
6. 初始线索池可新增线索、验证有效/无效、模拟分配销售、模拟申诉成功/失败
7. 渠道分析展示渠道综合评分和多维度对比
8. `npm run build` 构建成功
