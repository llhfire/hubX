# 项目管理底座模块设计

## 背景

现有系统是基于 React、Vite、Arco Design 的 CRM/OA 原型，已经有 `/projects` 和 `/projects/:id` 两个项目相关路由，但当前页面更偏项目执行、任务、团队、回款计划。`OA需求文档.md` 对 OA 1.0 第二阶段提出了项目模块规划，并明确后续会继续扩展项目成本核算、人工成本设置、报销、日报和投放日消耗等能力。

本设计先落地“项目管理底座”，为后续项目成本汇总和完整财务闭环打基础。第一版重点覆盖项目台账、项目详情、关联线索、关联日报、跟进记录和项目文档；成本核算只沉淀必要入口和工时数据，不在本阶段实现完整金额核算。

## 第一版范围

第一版为前端原型展示版：

- 改造现有项目列表 `/projects`
- 改造现有项目详情 `/projects/:id`
- 使用本地模拟数据展示项目、关联线索、关联日报、跟进记录和项目文档
- 支持项目新建、编辑、跟进、关联线索、解除线索关联、文档新增/编辑/删除等前端交互演示
- 关联客户按需求文档“暂不做”，仅保留占位说明和后续入口
- 不接后端接口
- 不做真实权限控制
- 不做真实数据持久化
- 不实现人工成本、报销、日消耗、利润和回款闭环，只预留后续成本核算入口

## 信息架构

### 项目列表

`/projects` 作为项目台账列表。

列表显示字段：

| 字段 | 说明 |
|------|------|
| 编号 | 项目编号 |
| 项目名称 | 点击进入项目详情 |
| 负责人 | 项目负责人，单选员工 |
| 销售 | 销售人员，可多选员工 |
| 对接主体 | 本公司主体 |
| 最新进展 | 最近一次项目跟进摘要 |
| 业务线 | 外包 / 自研 / 自运营 |
| 优先级 | 高 / 中 / 低 |
| 状态 | 未开始 / 进行中 / 已完成 / 验收中 / 搁置 / 延迟 / 催款中 |
| 开始日期 | 项目开始日期 |
| 预计结束日期 | 项目预计结束日期 |
| 总进度 | 0-100 百分比 |
| 添加时间 | 项目创建时间 |

搜索项：

- 项目名称
- 负责人
- 优先级
- 状态

列表操作：

- 详情：进入 `/projects/:id`
- 编辑：打开项目编辑弹窗
- 跟进：打开新增跟进弹窗
- 删除：确认后删除项目；若项目已有关联日报，提示确认影响

### 项目详情

`/projects/:id` 采用“左侧主内容 + 右侧协作栏”结构。

顶部摘要区展示：

- 项目名称
- 项目状态
- 总进度
- 负责人
- 预计结束日期
- 总工时
- 后续成本核算入口占位

左侧主内容使用 Tab：

1. 基础信息
2. 关联线索
3. 关联客户
4. 关联日报

右侧协作栏使用卡片或 Tab：

1. 跟进记录
2. 项目文档

这种布局让结构化关联信息集中在主区域，持续更新的信息集中在右侧，避免详情页过长。

项目基础信息。

字段：

- `id`
- `projectNo`
- `name`
- `latestProgress`
- `priority`: `高 | 中 | 低`
- `entity`
- `status`: `未开始 | 进行中 | 已完成 | 验收中 | 搁置 | 延迟 | 催款中`
- `businessLine`: `外包 | 自研 | 自运营`
- `salesUsers`
- `owner`
- `assistants`
- `productUsers`
- `uiUsers`
- `frontendUsers`
- `backendUsers`
- `progress`
- `startDate`
- `expectedEndDate`
- `remark`
- `attachments`
- `createdAt`

必填字段：

- 项目名称
- 优先级
- 状态
- 负责人

### ProjectFollowUp

项目跟进记录。

字段：

- `id`
- `projectId`
- `status`
- `progress`
- `content`
- `attachments`
- `operator`
- `createdAt`

新增跟进后，同步更新项目的：

- 最新进展
- 状态
- 总进度

### ProjectLeadRelation

项目关联线索。

字段：

- `id`
- `projectId`
- `leadNo`
- `leadName`
- `owner`
- `preSaleGroupName`
- `customerCategory`
- `source`
- `customerName`
- `phone`
- `wechat`
- `leadCreatedAt`

操作：

- 搜索线索并关联到项目
- 删除关联，只解除项目与线索的关系，不删除线索本身

### ProjectDailyReport

项目关联日报明细。

字段：

- `id`
- `projectId`
- `date`
- `projectName`
- `personName`
- `position`
- `hours`
- `workContent`
- `riskFeedback`

关联日报页签先展示人员工时汇总，再展示日报明细。

人员工时汇总字段：

- 编号
- 人员名称
- 职位
- 已用工时

汇总表最后一行展示总计工时。

### ProjectDocument

项目文档。

字段：

- `id`
- `projectId`
- `title`
- `onlineUrl`
- `owner`
- `uploadedFileName`
- `description`
- `createdAt`

必填字段：

- 标题
- 负责人

`onlineUrl` 和 `uploadedFileName` 至少填写一个。

## 核心交互

### 新建和编辑项目

新建/编辑项目使用弹窗表单。

表单字段按需求文档的项目模块属性组织，优先覆盖项目名称、最新进展、优先级、对接主体、状态、业务线、销售人员、负责人、协助人、UI、前端、后端、总进度、开始日期、预计结束日期和备注。

表单校验：

- 项目名称必填
- 优先级必填
- 状态必填
- 负责人必填
- 总进度限制在 0-100

### 新增跟进

项目列表和详情页都可触发新增跟进。

新增跟进字段：

- 状态
- 总进度
- 跟进详情
- 附件上传

提交后：

- 在跟进记录中新增一条记录
- 同步项目最新进展为本次跟进详情摘要
- 同步项目状态
- 同步项目总进度

### 关联线索

项目详情的“关联线索”页签提供“关联线索”按钮。

操作流程：

1. 点击“关联线索”打开弹窗
2. 输入标题或关键字搜索线索
3. 选择线索
4. 点击确认后追加到项目关联线索列表

线索列表字段按需求文档展示：编号、线索名称、归属人、售前群名称、客户分类、线索来源、客户称呼、联系电话、微信、线索添加时间和操作。

删除关联时弹出确认框，确认后只解除关系。

### 关联客户

第一版不实现客户绑定流程。

关联客户页签展示占位说明：客户关联将在后续客户模块联动阶段实现。页面可保留一个禁用或提示型入口，避免用户误以为功能已完成。

### 关联日报

项目详情的“关联日报”页签只读展示已经关联到当前项目的日报数据。

布局顺序：

1. 人员工时汇总表
2. 日报明细表

日报明细字段：

- 日期
- 项目名称
- 人员
- 耗时
- 工作内容
- 风险/异常反馈

新增日报仍通过现有日报模块完成，不在项目详情中重复实现日报新增入口。

### 项目文档

右侧“项目文档”区域提供新增文档按钮。

新增/编辑文档字段：

- 标题
- 线上地址
- 负责人
- 上传文档
- 文档说明

文档列表字段：

- 编号
- 标题
- 线上地址
- 负责人
- 文档下载
- 文档说明
- 添加日期
- 操作：编辑、删除

上传文档第一版只模拟文件名保存，不做真实文件上传。

## 成本核算入口

本阶段只为后续成本核算沉淀入口和数据基础。

项目详情顶部展示：

- 总工时
- 人员工时汇总入口
- 成本核算占位说明

后续第二阶段可基于以下数据扩展项目成本汇总：

- 日报工时
- 人工成本设置
- 项目报销
- 投放日消耗
- 项目回款和利润

第一版不计算人工成本、报销合计、投放消耗合计、项目利润率或回款状态。

## 错误处理

只处理前端原型阶段真实会遇到的业务边界：

- 必填字段缺失时阻止提交并显示表单错误
- 总进度低于 0 或高于 100 时阻止提交
- 文档未填写线上地址且未上传文件时阻止提交
- 删除项目、删除关联线索、删除文档前使用确认弹窗
- 已有关联日报的项目执行删除时提示确认影响
- 关联客户页签明确标注第一版暂不实现

不处理真实接口错误、真实上传失败、权限鉴权失败或并发冲突，这些留到后端接口阶段。

## 测试与验证

构建验证：

```bash
npm run build
```

浏览器验证：

1. 启动 Vite 开发服务器
2. 打开 `/projects`
3. 打开 `/projects/1`

Golden path：

1. 项目列表展示需求文档指定字段
2. 使用项目名称、负责人、优先级、状态进行筛选
3. 新建项目，必填项填写后可提交并出现在列表中
4. 编辑项目，保存后列表和详情同步展示新值
5. 在列表或详情新增跟进，项目最新进展、状态和总进度同步更新
6. 进入项目详情查看基础信息
7. 在关联线索页签搜索并关联线索
8. 删除线索关联，只解除关联关系
9. 在关联日报页签查看人员工时汇总和日报明细
10. 在右侧新增项目文档
11. 编辑项目文档
12. 删除项目文档

边界验证：

1. 新建项目缺少项目名称、优先级、状态或负责人时不能提交
2. 总进度小于 0 或大于 100 时不能提交
3. 新增文档时标题或负责人为空不能提交
4. 新增文档时线上地址和上传文档都为空不能提交
5. 删除确认弹窗点击取消后数据保持不变
6. 关联客户页签显示暂不实现说明
7. 有关联日报的项目删除时出现影响提示

## 验收标准

1. `/projects` 作为项目台账列表，字段与需求文档一致
2. 项目列表支持项目名称、负责人、优先级、状态筛选
3. 项目列表操作包含详情、编辑、跟进、删除
4. `/projects/:id` 详情页包含基础信息、关联线索、关联客户、关联日报
5. 详情页右侧包含跟进记录和项目文档
6. 新增跟进后同步项目最新进展、状态和总进度
7. 关联线索支持搜索选择、确认关联、删除关联
8. 关联日报展示人员工时汇总和日报明细
9. 项目文档支持新增、编辑、删除
10. 关联客户明确标注第一版暂不实现
11. 成本核算只展示工时和后续入口，不实现完整金额核算
12. `npm run build` 构建成功
