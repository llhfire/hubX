# 公司主体信息维护与线索入口查看设计

## 背景

现有系统已有“本公司主体管理”页面，当前支持公司主体列表、新建、编辑、查看和删除。现有字段集中在基础工商/银行信息，弹窗为单表单结构。用户希望扩展为三类信息维护，并允许在线索列表的公司主体字段中打开公司信息弹窗查看。

## 目标

- 在本公司主体管理中维护公司信息。
- 公司信息按三个 Tab 展示：基础信息、开票信息、公司资料。
- 公司资料分为不可编辑 PDF 和可编辑 PPTX 两类，原型阶段只展示文件列表与按钮占位，不接真实上传。
- 维护公司信息需要权限控制。
- 在线索列表点击公司主体名称时打开同一公司信息弹窗，但只读展示，并默认打开“公司资料”Tab。

## 非目标

- 不接真实后端接口。
- 不实现真实文件上传、预览或下载。
- 不实现刷新后持久化。
- 不在线索列表中直接编辑公司主体信息。

## 推荐方案

采用“维护与查看分离 + 复用同一弹窗组件”的方案：

- 本公司主体管理页负责新建、编辑、删除、资料维护。
- 线索列表只提供公司主体详情查看入口。
- 抽取一个可复用的公司主体三 Tab 弹窗组件，通过模式控制只读或编辑。
- 线索入口打开弹窗时默认选中“公司资料”Tab；管理页入口默认选中“基础信息”Tab。

这样可以保持两个入口展示一致，同时避免在线索业务页面误改主体资料。

## 信息结构

### 基础信息

基础信息沿用现有公司主体字段：

- 公司名称
- 公司简称
- 统一社会信用代码
- 法定代表人
- 注册资本
- 注册地址
- 联系电话
- 状态

### 开票信息

开票信息参考现有客户开票信息字段，作为独立 Tab 维护：

- 开票抬头
- 纳税人识别号
- 开户银行
- 银行账号
- 开票地址
- 开票电话

开票抬头和纳税人识别号可默认与公司名称、统一社会信用代码一致，但允许单独维护。

### 公司资料

公司资料分两类展示：

- PDF 资料：用于方案和报价等场景的不可编辑资料，只显示查看/下载按钮占位。
- PPTX 资料：用于方案和报价等场景的可编辑模板资料，显示下载/替换按钮占位。

原型阶段使用 mock 文件列表，不接真实上传。无资料维护权限时，不显示替换/上传类按钮。

## 组件设计

### CompanyEntityInfoModal

新增可复用弹窗组件，负责展示和编辑公司主体信息。

核心输入：

- `visible`：弹窗显示状态。
- `record`：当前公司主体数据。
- `mode`：`view` 或 `edit`。
- `defaultTab`：默认激活 Tab，支持 `basic`、`invoice`、`files`。
- `permissions`：当前用户对公司主体的查看、编辑、删除、资料维护权限。
- `onOk` / `onCancel`：保存和关闭回调。
- `onGoManage`：线索入口中跳转到公司主体管理页的回调。

行为：

- `mode=view` 时所有字段只读。
- `mode=edit` 时显示表单字段，允许编辑基础信息、开票信息和资料维护按钮。
- 管理页调用时 `defaultTab=basic`。
- 线索列表调用时 `defaultTab=files`。

## 页面改造

### 本公司主体管理页

保留现有列表结构，并调整弹窗入口：

- “新建主体”：打开 `CompanyEntityInfoModal` 编辑模式，默认基础信息 Tab。
- “编辑”：打开编辑模式，带入当前主体数据。
- “查看”：打开只读模式，默认基础信息 Tab。
- “删除”：保留现有二次确认。
- 新增/编辑/删除/资料维护按钮根据权限显示或禁用。

保存时校验基础必填字段和开票必填字段，成功后提示并关闭弹窗。原型阶段数据仍使用本地 mock 状态或静态 mock 数据。

### 线索列表

在线索列表的公司主体字段上增加点击入口：

- 公司主体名称渲染为链接样式文本。
- 点击后检查查看权限。
- 有查看权限：打开 `CompanyEntityInfoModal` 只读模式，默认选中“公司资料”Tab。
- 无查看权限：提示无权限查看公司主体详情。
- 有维护权限时，弹窗底部显示“去公司主体管理维护”按钮。
- 点击“去公司主体管理维护”跳转到 `/system/company`。

## 权限设计

现有权限树中已有 `system-company`。设计上补充细分权限项：

- `system-company-view`：查看公司主体详情。
- `system-company-create`：新建公司主体。
- `system-company-edit`：编辑公司主体基础信息和开票信息。
- `system-company-delete`：删除公司主体。
- `system-company-files`：维护公司资料。

原型阶段仍使用本地 mock 权限，不接后端。权限效果：

- 无查看权限：不可打开详情。
- 有查看权限但无维护权限：可查看三 Tab，不显示维护按钮。
- 有维护权限：可在公司主体管理页执行对应维护动作。

## 数据流

1. 公司主体列表从页面 mock 数据读取。
2. 用户在管理页点击新建/编辑/查看，页面将当前记录、模式、默认 Tab 和权限传入弹窗组件。
3. 用户在线索列表点击公司主体名称，线索页按主体名称或主体 ID 找到对应 mock 公司主体记录，并以只读模式打开弹窗。
4. 弹窗内部按模式渲染表单或只读信息。
5. 保存时由管理页处理 mock 数据更新和提示。

## 错误与边界处理

- 线索中的公司主体未匹配到记录：提示“未找到公司主体信息”。
- 无查看权限：提示“暂无权限查看公司主体详情”。
- 无维护权限：隐藏或禁用新建、编辑、删除、替换资料按钮。
- 必填字段缺失：使用 Arco Form 校验提示。
- 公司资料为空：展示空状态和说明文案。

## 测试与验收

- 公司主体管理页可以查看、新建、编辑三 Tab 信息。
- 管理页打开弹窗时默认显示基础信息 Tab。
- 公司资料 Tab 能区分展示 PDF 和 PPTX 两类 mock 文件。
- 线索列表点击公司主体名称后打开只读弹窗。
- 线索入口打开时默认显示公司资料 Tab。
- 线索弹窗中不可直接编辑公司主体信息。
- 有维护权限时线索弹窗显示“去公司主体管理维护”入口。
- 无查看权限时点击公司主体名称提示无权限。
- 构建通过。
