# 日报管理-销售人员差异化日报模板 设计文档

**日期**: 2026-05-14
**状态**: 已批准

---

## 1. 背景与目标

当前系统所有用户使用统一的日报模板（项目任务 + 今日总结 + 明日计划）。需求为销售人员设计专门的日报模板，并支持其他用户自定义日报模板。

**核心目标**：
- 销售人员使用专属日报模板，包含"线索跟进情况"等字段
- 通用用户使用标准模板，包含"遇到的问题"字段
- 用户可覆盖角色默认模板设置

## 2. 数据模型

### 2.1 日报模板（DailyTemplate）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 主键 |
| name | string | 模板名称，如"销售日报模板"、"通用日报模板" |

| fields | Field[] | 字段列表，JSON格式存储 |
| isDefault | boolean | 是否为该类型的默认模板 |
| createdAt | datetime | 创建时间 |

每个模板的字段配置，存储在 `DailyTemplate.fields` 中：

| 字段 | 类型 | 说明 |

| type | enum | `text`, `textarea`, `lead-tracking`, `project-task-list`, `date`, `select` |

| 字段 | 类型 | 说明 |

## 4. 功能流程

### 4.1 销售人员日报提交流程

1. 销售人员点击顶部导航的日报入口打开模态框
2. 系统根据用户配置（个人覆盖 > 角色默认）确定使用的模板
3. 对于销售模板，自动从线索跟进记录中汇总当天数据：
   - 列出当天有跟进的所有线索，按级别排序（S > A > B > C）
   - 显示每条线索当天的状态变更（如：意向低 → 意向高）
   - 显示每条线索当天的跟进详情记录
4. 销售人员可编辑预填数据，填写"需协助事项"和"明日工作计划"
5. 提交时校验必填项
6. 提交后关闭模态框

### 4.2 线索跟进数据自动汇总逻辑

```typescript
function getSalesDailyLeadsData(userId: string, date: Date) {
  // 1. 获取当日该销售跟进的所有线索
  const leads = getLeadsFollowedByUserToday(userId, date);

  // 2. 按级别排序
  leads.sort((a, b) => {
    const levelOrder = { S: 0, A: 1, B: 2, C: 3 };
    return levelOrder[a.level] - levelOrder[b.level];
  });

  // 3. 对每条线索获取当日变更和跟进记录
  return leads.map(lead => ({
    leadId: lead.id,
    leadName: lead.name,
    level: lead.level,
    statusChanges: getStatusChangesToday(lead.id, date),
    followRecords: getFollowRecordsToday(lead.id, date),
  }));
}
```

### 4.3 通用用户日报提交流程

1. 用户点击顶部导航的日报入口打开模态框
2. 系统根据用户配置确定使用通用模板
3. 用户填写：项目任务（可多行）、今日总结、遇到的问题、明日工作计划
4. 提交时校验必填项
5. 提交后关闭模态框

### 4.4 日报评论流程

1. 任意可见用户（日报提交者或被@者）在日报详情页点击评论入口
2. 输入评论内容，可@其他人（输入@后弹出用户选择）
3. 提交评论后：
   - 评论关联的日报的作者和被@用户收到通知
   - 评论写入数据库，mentionedUsers 记录被@用户
4. 被@用户或未读评论的用户：
   - 日报菜单右侧显示红点badge提示
   - 进入日报列表/详情后可看到评论

## 7. 技术实现要点

### 7.1 模板渲染策略

```typescript
// 根据用户配置获取模板
function getUserTemplate(userId: string): DailyTemplate {
  // 1. 优先检查用户个人模板配置
  const userConfig = getUserTemplateConfig(userId);
  if (userConfig?.templateId) {
    return getTemplateById(userConfig.templateId);
  }

  // 2. 否则根据用户角色获取默认模板
  const userRole = getUserRole(userId);
  if (userRole === 'sales') {
    return getDefaultSalesTemplate();
  }
  return getDefaultGeneralTemplate();
}
```

### 7.2 预填数据处理

销售日报的线索跟进情况由系统自动预填，用户可编辑：

- 预填数据只作为初始值，不影响原线索记录
- 用户编辑后的数据仅存储在该日报的 content 中
- 提交时将最终 content 存入数据库
