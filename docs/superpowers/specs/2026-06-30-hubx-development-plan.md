# HubX 开发计划

## 概述

基于 HubX OA 系统的现有基础，规划 **4 个扩展 + 11 个新模块**，分四个阶段实施。

> 最新更新：2026-06-30 — 新增「员工能力建模」模块，合并入第一阶段。

---

## 第一阶段：基础夯实

为后续所有模块提供底座数据（员工、日报工时、能力数值）。

### 1.1 员工管理（新增）

**依赖**：无

**范围**：
- 员工档案：基本信息、部门、职位、职级、入职日期、转正日期、合同到期日
- 入离职流程：入职审批、转正提醒、离职交接
- 考勤管理：请假/加班/调休
- 绩效考核：月度/季度考核评分
- 职级时薪设置：按职级维护标准时薪（为项目成本核算提供数据）

**规格文档**：待编写

### 1.2 员工能力建模（新增）

**依赖**：员工管理（员工基础信息、职级、部门）

**范围**：
- 五维能力值系统：技术/业务/管理/工具/领域（0-100）
- 职位权重映射：不同职位对五维能力的权重配置
- L1-L10 等级体系：每个职位独立的等级阶梯
- 技能树：最多3层（基础→进阶→专家），前置依赖解锁 + 项目实践触发，允许跨岗位
- 经验获取：日报驱动 + 项目结算 + 主管评定，三通道
- 晋级流程：数值达标 → 解锁资格 → 评审审批 → 晋升
- 三张画像：面试画像 → 入职画像 → 在职画像（含版本管理）
- 履历管理：入职前项目录入 + 在职项目自动同步
- 招聘量化：面试评分卡 → 主观印象转客观数据 → 面试画像生成

**规格文档**：`docs/superpowers/specs/2026-06-30-employee-capability-modeling-design.md`

### 1.3 个人工作台（扩展）

**依赖**：员工能力建模

**范围**：
- 个人仪表盘：今日待办、我的项目、我的日报、提醒
- 任务清单：个人任务/团队任务，优先级排序
- 能力面板卡片：等级进度条、五维雷达图、下一个可解锁技能
- 快捷入口：填写日报、新建线索、查看合同

**规格文档**：待编写

### 1.4 日报模块扩展

**依赖**：员工管理（角色/部门）、项目管理（关联项目）

**范围**：
- 新增投放日报模板（平台/账户/消耗/展示/点击/线索）
- 扩展通用日报模板（增加工作种类字段）
- 统一规范：所有日报必须包含工作种类 + 项目 + 工时
- 工作种类 → 能力维度映射（为员工能力建模提供经验输入）

**规格文档**：待编写

## 第二阶段：核心业务闭环

打通「线索→合同→项目→成本→回款→利润」的核心价值链。

### 2.1 项目成本核算（新增）

**依赖**：员工管理（职级时薪）、日报扩展（工时数据）

**范围**：
- 人工成本设置：按职级维护标准时薪（仅财务可见）
- 项目成本 = Σ(人员工时 × 职级时薪) + 报销成本
- 项目利润 = 合同金额 − 项目成本
- 成本看板：按项目/部门/时间段查看成本与利润
- 预算预警：成本超预算自动提醒

**规格文档**：待编写

**ADR**：`docs/adr/0001-project-cost-job-level-rate.md`

### 2.2 合同回款（扩展）

**依赖**：合同管理

**范围**：

**A. 全合同回款 Kanban 看板**
- 布局：五列 Kanban — `正常回款 | 即将到期 | 已逾期 | 卡点阻塞 | 已结清`
- 顶部摘要栏（7 指标）：总合同数、总应收金额、本月已回款、逾期金额、卡点合同数、卡点总金额、预计本月回款
- 合同卡片（10 项信息）：合同编号+客户名、合同金额、回款进度条、卡点标签、下期待付金额+日期、负责人头像、最近催款时间、卡点已持续天数
- 筛选：按负责人、客户、日期范围、卡点类型筛选
- 拖拽：支持拖动合同卡片切换状态列

**B. 侧边抽屉详情面板**
- 回款时间线：付款节点（计划）+ 实际到账记录 + 卡点事件，时间轴可视化
- 合同文本：展示合同正文（复用现有合同模板渲染）
- 快捷操作：标记催款、解决卡点、录入回款

**C. 卡点类型（5 种）**
- 逾期未付：过了付款节点未到账
- 客户拖延：已催款但客户无响应
- 开票未回：发票已开出但款未到
- 验收卡住：项目未验收导致付款节点无法触发
- 合同纠纷：有争议导致暂停付款

**D. 单合同回款看板优化**
- 回款计划 vs 实际回款对比
- 逾期标记与催款提醒
- 付款节点甘特图

**E. 回款预测**
- 按合同付款节点预测未来 3/6/12 个月现金流
- 逾期风险预警（即将到期/可能逾期）

**规格文档**：待编写

### 2.3 线索治理（扩展）

**依赖**：线索管理

**范围**：
- 跟进达标规则：每日/每周最低跟进次数、线索响应时间
- 动作规范：必须填写跟进记录、必须更新线索状态
- 违规限制：超时未跟进 → 自动回收线索、限制领取新线索数量
- 治理看板：各销售达标率、违规次数、线索转化率

## 第三阶段：运营扩展

完善外包业务的周边配套模块。

### 3.1 售后运维（新增）

**依赖**：项目管理（已交付项目）、合同管理（质保条款）

**范围**：
- 维护期跟踪：项目交付后自动进入免费维护期，到期提醒
- 客户工单：问题提交、分配、处理、关闭，SLA 时效
- 付费运维合同：维护期满后签约续费
- 续费提醒：到期前自动通知销售/客服

**规格文档**：待编写

### 3.2 供应商管理（新增）

**依赖**：项目管理（分包需求）

**范围**：
- 供应商档案：公司/个人、技能标签、合作评级
- 分包合同：关联项目、金额、付款节点
- 付款记录：已付/未付，关联财务
- 项目关联：哪些项目分包给了哪个供应商

**规格文档**：待编写

### 3.3 会议管理（新增）

**依赖**：员工管理（参会人员）

**范围**：
- 会议安排：主题、时间、地点、参会人、议程
- 会议记录：决议事项、行动项、负责人、截止日期
- 会议提醒：开始前通知参会人
- 关联项目/客户：会议可关联到具体项目或客户

**规格文档**：待编写

### 3.4 知识库（新增）

**依赖**：无

**范围**：
- 文档管理：创建/编辑/分类/搜索
- 模板库：技术方案模板、报价模板、SOP 流程文档
- 项目复盘：项目结束后自动提示写复盘文档
- 权限控制：公开/部门/项目级可见性
- 标签系统：按技术栈、行业、项目类型分类

**规格文档**：待编写

### 3.5 资产管理（新增）

**依赖**：无

**范围**：
- IT 资产登记：服务器、域名、SSL 证书、设备、软件许可证
- 到期提醒：域名续费、SSL 续期、许可证续费
- 资产分配：设备分配给谁、归还记录
- 资产盘点：定期盘点核对

**规格文档**：待编写

## 第四阶段：智能化

数据贯通后的高阶分析 + AI 自动化。

### 4.1 全链路 ROI（新增）

**依赖**：线索成本、线索管理、合同管理、项目成本核算

**范围**：
- 漏斗分析：投放消耗 → 线索数/成本 → 客户转化率 → 合同金额 → 项目利润
- 渠道 ROI：按百度/小红书/抖音/淘宝拆分 ROI
- 人员 ROI：按销售人员拆分线索转化率与合同金额
- 项目 ROI：按项目类型拆分利润率

**规格文档**：待编写

### 4.2 AI 驱动（新增）

**依赖**：合同管理、项目管理、员工能力建模、会议管理

**范围**：
- 合同 → 自动拆解任务（根据合同 SOW 自动生成项目任务）
- 自动分配人员（根据技能树和当前负载匹配最佳人员）
- 自动项目跟进（根据交付计划自动提醒关键节点）
- 自动安排会议（根据项目阶段自动预约评审/验收会议）

**规格文档**：待编写

## 模块依赖图

```
第一阶段:
  员工管理 ──┬── 员工能力建模 ── 个人工作台
             │
             └── 日报扩展
             
第二阶段:
  员工管理 ──┐
  日报扩展 ──┼── 项目成本核算
             │
  合同管理 ── 合同回款
  线索管理 ── 线索治理

第三阶段:
  项目管理 ──┬── 售后运维
             ├── 供应商管理
  员工管理 ── 会议管理
  (独立)  ── 知识库
  (独立)  ── 资产管理

第四阶段:
  线索成本+线索+合同+成本 ── 全链路ROI
  合同+项目+能力建模+会议 ── AI驱动
```
