# 合同成本统计与工资维护设计

## 背景

现有财务统计中合同统计 Tab 的成本数据（科研、商务、外包、其他）为硬编码数字，无法追溯明细。系统缺少工资管理能力，无法基于人员工时和薪资计算真实科研成本。用户希望：建立工资表、关联合同与项目、从日报工时计算科研成本、从报销归集商务成本、分摊运营成本，并在合同维度展示完整成本明细。

## 目标

- 新增工资表页面，支持按月查看和管理员工名义/实际工资与工时。
- 建立合同↔项目双向关联，打通"合同→项目→日报工时→薪资"的科研成本计算链路。
- 改造合同统计，点击合同名称跳转到独立成本明细页面，展示科研、商务、外包、其他、分摊运营五类成本。
- 通过权限控制工资数字的可见性，仅财务和人事角色可查看。

## 非目标

- 不接真实后端接口。
- 不实现真实 Excel 解析，导入按钮只做界面占位。
- 不实现刷新后持久化。
- 不改造组织架构 UI，只在 mock 数据中补充员工入离职时间。

## 模块结构

### 1. 工资表页面

路径：`/finance/salary`

财务管理菜单下新增"工资表"入口。

**页面功能：**

- 顶部月份选择器，切换查看不同月份。
- 表格列：员工姓名、部门、职位、名义月工资、名义月工时（系统默认 176h）、名义时薪（自动计算）、实际月工资、实际月工时、实际时薪（自动计算）。
- "导入工资"按钮和"导入实际工时"按钮：点击弹出上传弹窗，原型阶段只做界面占位。
- 若当月未录入工资，自动沿用上月数据，页面用标签提示"沿用上月"。
- 无工资查看权限时菜单不显示；有查看权限但无编辑权限时隐藏导入按钮。
- 无薪资查看权限的角色看到工资列显示"***"。

**名义 vs 实际：**

- 每个员工每月有两套数据：名义（月中可用于估算）和实际（月初由人事更新后生效）。
- 名义时薪 = 名义月工资 ÷ 名义月工时。
- 实际时薪 = 实际月工资 ÷ 实际月工时。
- 成本统计中：当月未更新实际数据时用名义时薪，更新后切换为实际时薪。

### 2. 合同↔项目关联

- 合同 mock 数据增加 `projectIds: string[]` 字段。
- 项目 mock 数据增加 `contractId: string` 字段。
- 原型阶段在 mock 数据中硬编码关联关系，不做 UI 层的关联编辑。

### 3. 员工入离职时间

- 员工 mock 数据增加 `leaveDate` 字段（离职日期，可选）。
- 原型阶段只在 mock 数据中增加，不改造组织架构 UI。

### 4. 合同统计改造

在现有财务统计→合同统计 Tab 中：

- 合同名称改为可点击链接样式。
- 点击后跳转到 `/finance/contract-cost/:contractId`。

### 5. 合同成本明细页面

路径：`/finance/contract-cost/:contractId`

**页面布局：**

顶部面包屑导航：财务统计 > 合同成本明细 > {合同名称}。

成本概览卡片区（一行 5 张）：科研成本、商务成本、外包成本、其他成本、分摊运营成本。每张卡片显示金额和"名义估算"/"实际"标签。

下方分 5 个 Tab 展示明细：

**Tab 1 - 科研成本：**
- 人员工时明细表：员工姓名、职位、关联项目、投入工时、时薪、小计金额。
- 表尾合计行。
- 顶部标注当前使用名义/实际时薪。

**Tab 2 - 商务成本：**
- 报销明细表：报销单号、报销人、费用类型（差旅/交通/招待）、金额、关联线索、日期。
- 按费用类型小计。

**Tab 3 - 外包成本：**
- 外包明细表：供应商、项目、金额、日期、备注。
- 表尾合计行。

**Tab 4 - 其他成本：**
- 其他明细表：类别、名称、金额、日期、备注。
- 表尾合计行。

**Tab 5 - 分摊运营成本：**
- 展示计算过程：当月运营总费用、在职员工总数、标准工时、每小时运营成本。
- 该合同分摊明细：员工姓名、在该合同项目投入工时、分摊金额。
- 表尾合计行。

## 成本计算公式

### 科研成本

```
某员工在某合同的科研成本 = 该员工在该合同所有关联项目的日报工时合计 × 时薪
时薪 = 名义时薪（月中）或 实际时薪（月初更新后）
名义时薪 = 名义月工资 ÷ 名义月工时
实际时薪 = 实际月工资 ÷ 实际月工时
合同科研成本 = 所有相关员工的科研成本之和
```

### 商务成本

```
通过 报销单→线索→客户→合同 链路归集。
按差旅费、交通费、招待费分类汇总。
合同商务成本 = 归集到该合同的所有报销金额之和。
```

### 外包成本

直接从合同关联的外包记录中汇总金额。

### 其他成本

直接从合同关联的其他成本记录中汇总金额。

### 分摊运营成本

```
每小时运营成本 = 当月运营费用总额 ÷ 当月在职员工总数 ÷ 当月标准工时
该合同分摊运营成本 = 该合同下所有人员在该合同项目的总工时 × 每小时运营成本
```

## 权限设计

在权限树"财务管理"下新增：

1. `finance-salary-view`：查看工资表（仅财务和人事默认拥有）。
2. `finance-salary-edit`：编辑/导入工资和实际工时（仅人事默认拥有）。
3. `finance-contract-cost-view`：查看合同成本明细页面。
4. `finance-contract-cost-detail`：查看成本明细中的人员薪资数字（仅财务和人事）。

权限效果：

- 无 `finance-salary-view`：工资表菜单不显示。
- 有 `finance-salary-view` 但无 `finance-salary-edit`：可查看但无导入按钮。
- 无 `finance-contract-cost-view`：合同统计中合同名称不可点击。
- 有 `finance-contract-cost-view` 但无 `finance-contract-cost-detail`：科研成本 Tab 中时薪和金额列显示"***"，只能看到工时。

原型阶段使用本地 mock 权限，默认全部开启。

## 数据流

1. 工资表数据从 mock 状态读取，包含每月每人的名义和实际工资/工时。
2. 合同成本明细页从 mock 合同数据获取 projectIds，遍历关联项目的人员和日报工时。
3. 科研成本通过 合同→项目→日报工时→工资表 链路计算。
4. 商务成本通过 报销→线索→客户→合同 链路归集。
5. 外包和其他成本直接从合同维度的 mock 记录读取。
6. 运营成本从现有运营成本 Tab 数据计算每小时运营成本，再按合同工时占比分摊。

## 涉及文件改动范围

- 新增：工资表页面、合同成本明细页面、共享成本计算模块。
- 改造：财务统计合同统计 Tab（合同名称可点击跳转）、路由、侧边栏菜单、权限树。
- 补充：合同 mock 数据加 projectIds、项目 mock 数据加 contractId、员工 mock 数据加 leaveDate。

## 错误与边界处理

- 合同无关联项目：科研成本和分摊运营成本显示为 0，提示"该合同未关联项目"。
- 员工无工资数据：该员工的科研成本按 0 计算。
- 当月无运营费用数据：分摊运营成本显示为 0。
- 无查看权限：合同名称不可点击；工资表菜单不显示。
- 无薪资查看权限：工资列和时薪列显示"***"。

## 测试与验收

- 工资表页面可切换月份查看不同月份数据。
- 工资表"导入工资"和"导入实际工时"按钮弹出上传占位弹窗。
- 未录入当月工资时显示"沿用上月"标签。
- 合同统计中合同名称可点击，跳转到成本明细页面。
- 成本明细页面 5 张概览卡片金额正确。
- 科研成本 Tab 显示人员工时明细，金额由工时×时薪正确计算。
- 商务成本 Tab 按差旅/交通/招待分类展示报销明细。
- 外包和其他成本 Tab 展示明细表。
- 分摊运营成本 Tab 展示计算过程和按人员分摊明细。
- 权限控制生效：工资数字对无权限角色显示"***"。
- 构建通过。
