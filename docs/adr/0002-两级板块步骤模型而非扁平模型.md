# 0002-两级板块步骤模型而非扁平模型

## 状态

已接受

## 背景

SOP 生成的任务列表需要组织 7 大板块、30+ 个步骤。需要决定数据模型采用扁平结构还是两级结构。

## 决策

采用两级模型：`SopPhase`（板块）+ `SopStep`（步骤），板块是步骤的父实体。

## 理由

核心驱动因素是业务需求：**不同板块有不同的主管**。板块主管和步骤执行人是两个不同层级的责任人：
- 板块主管负责整个板块的协调和进度把控
- 步骤执行人负责具体步骤的执行

扁平模型无法自然表达"板块主管"这个角色——如果用字段标注，板块行的状态、日期都需要从步骤聚合计算，本质上就是在扁平结构上模拟两级关系。

## 后果

- 优势：板块主管有独立归属，甘特图可按板块折叠/展开，板块状态自动推导逻辑清晰
- 风险：两级结构增加了状态管理的复杂度（需同步 phaseId、联动板块状态），在纯 useState 架构下需要手动维护一致性
