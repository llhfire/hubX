# 0005-工作项关联SOP步骤采用提示联动而非自动联动

工作项可以关联到交付计划的 SOP 步骤。当 SOP 步骤在交付计划中被标记为"已完成"时，采用提示联动（询问用户是否将关联的工作项也标记为完成），而非自动联动或完全独立。

## 状态

已接受

## 背景

工作项与 SOP 步骤的关联是项目模块的核心集成点。SOP 步骤完成时，关联的工作项状态如何处理存在三种选择：自动联动、提示联动、完全独立。

## 决策

采用提示联动：SOP 步骤完成时，提示"关联的 N 个任务是否也标记为已完成？"，用户一键确认或跳过。

## 理由

- 自动联动太强硬：SOP 步骤完成不等于工作项完成（步骤可能完成了但工作项还有收尾工作）
- 完全独立太松散：容易遗漏，用户需要手动逐个检查
- 提示联动兼顾效率和准确性，给用户控制权

## 后果

- 优势：减少手动操作的同时保证数据准确性
- 风险：用户可能习惯性点"确认"导致误标记，但操作历史会记录变更可追溯
