# ADR-0001: 项目成本核算采用职级标准时薪

## 状态

已采纳

## 背景

项目成本核算需要计算人力成本：人员工时 × 时薪。时薪的设定方式有三种选择：

- **A）个人时薪**：财务为每个员工单独设定时薪，灵活但维护成本高，且可能引发内部薪资对比
- **B）项目时薪**：同一人在不同项目可有不同时薪，灵活但极其复杂
- **C）职级时薪**：按职位/职级设标准时薪（如 JAVA 工程师 80元/小时、UI 设计师 70元/小时）

## 决策

选择 **C — 按职级设定标准时薪**。

## 理由

1. **维护成本低**：公司有几十个员工但只有几个职级，财务只需维护职级-时薪映射表
2. **项目报价一致**：外包报价通常也是按角色/人天报，职级时薪与报价逻辑一致
3. **避免敏感信息扩散**：项目成本看板可能被项目经理等角色看到，职级时薪不会暴露个人薪资
4. **够用原则**：对于项目利润核算（而非精确到个人的成本归集），职级级别的精度已经足够

## 影响

- 员工管理模块需维护「职级」字段，且职级与项目中的角色（JAVA/PHP/前端/UI 等）对齐
- 人工成本设置模块按职级而非按人维护时薪
- 项目成本核算公式：`Σ (人员日报工时 × 人员所属职级的标准时薪)`
- 如需精确到个人的成本核算，未来可升级为个人时薪覆盖职级时薪（保留扩展空间）
