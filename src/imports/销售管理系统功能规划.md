# 销售管理系统功能规划

## 一、规划概述

本规划旨在构建一套覆盖**从线索获取到项目交付、资金回款**全业务链路的项目型销售管理系统（CRM）。系统将打破传统 CRM 仅关注销售前端的局限，深度融合项目管理能力，实现销售业务与交付业务的数字化闭环，解决线索跟进不及时、项目执行不透明、成本核算不精细等核心痛点。

系统核心遵循 **L2C \(Leads to Cash\)** 业务逻辑，将用户的业务诉求拆解为 “基础支撑、客户管理、线索商机、合同交易、项目交付、数据洞察” 六大业务域，实现业务全流程的数字化管控。

## 二、整体业务流程

```Plain Text
线索获取 → 公海池流转 → 销售认领 → 线索跟进 → 商机培育 → 合同签订 → 项目启动 → 任务执行 → 产物交付 → 回款结算 → 数据复盘
```

整个流程中，**一条线索即一个项目**，所有业务数据围绕该主体进行沉淀，实现从销售前端到交付后端的全生命周期数据拉通。

## 三、核心功能模块设计

### 3\.1 基础支撑模块：组织与权限

作为系统的运行底座，支撑整个业务体系的人员与权限管控。

- **组织架构管理**

    - 支持按企业实际架构创建部门、人员信息。

    - 支持角色定义与权限分配，实现不同岗位的数据与操作权限隔离。

- **基础数据字典**

    - 统一维护线索来源、客户行业、意向标签、跟进方式等基础枚举数据，支持后台自定义配置。

### 3\.2 客户全生命周期管理

构建统一的客户 360° 视图，沉淀客户全量业务数据。

- **客户主体管理**

    - 支持创建个人、企业、机构等多类型客户主体。

    - 客户资料覆盖完整维度：

        1. **基础信息**：主体名称、类型、所属行业、规模、注册资本、地址、统一社会信用代码等。

        2. **联系人管理**：支持维护多联系人，标记默认联系人，记录电话、微信等全渠道联系方式。

        3. **业务属性**：客户来源、客户等级、状态、预计销售额、历史合同数 / 金额、待收款金额、信用等级。

        4. **财务信息**：开票信息、汇款账户信息等。

- **客户操作入口**

    - 从客户详情页可直接编辑信息、查看历史线索 / 项目、新建线索、导出客户资料。

### 3\.3 线索与商机管理

作为销售业务的起点，实现线索的精细化流转与跟进。

#### 3\.3\.1 公海线索池

- **公海机制**：未被认领的线索统一进入公海池，全公司销售可见。

- **线索创建**：支持任何人新建公海线索，录入基础信息。

- **公海操作**：销售可对线索进行认领、查看详情、标记为垃圾线索。

#### 3\.3\.2 线索核心字段

每条线索承载完整的前端销售信息，核心字段包括：

|字段分类|具体字段|
|---|---|
|来源信息|线索来源（百度推广、抖音、小红书等）、推广关键词|
|基础信息|线索名称、联系人、联系人手机号、客户需求梗概|
|业务标签|意向标签（APP、小程序、管理系统等）、意向等级（高 / 中 / 低）|
|关联信息|关联客户主体、对接公司主体|
|时间信息|创建时间、最后更新时间|

#### 3\.3\.3 认领后线索管理

线索被认领后，转为销售私有线索，支持以下操作：

- **线索流转**：

    - **转让**：销售可将线索转让给同事，权限同步转移。

    - **放弃 / 抛回**：可将线索放弃，重新抛回公海池。

    - **标记垃圾**：可将无效线索标记为垃圾，停止流转。

- **智能提醒机制**：

    - 销售认领线索后，第 7 天起每日提醒跟进，直至线索签单或被抛回公海。

    - 支持设置下次跟进提醒，提供默认 \(3 天\)、高频 \(1 天\)、低频 \(10 天\) 三种快捷选项。

#### 3\.3\.4 线索跟进记录

精细化记录每一次客户互动，沉淀跟进过程数据。

- **跟进信息录入**：

    - 跟进时间、跟进方式（电话、微信、上门拜访等）。

    - 客户状态流转：未联系、未接通、初步沟通、需求调研、方案报价、合同洽谈、已签单、已终止。

    - 客户意向更新、标签更新。

    - 富文本跟进详情，支持记录沟通细节、客户反馈。

- **跟进日志沉淀**：所有跟进记录自动按时间倒序沉淀，支持回溯完整跟进历史。

### 3\.5 项目执行与交付管理

针对项目型销售的核心模块，实现合同执行过程的数字化、分子化管控。

#### 3\.5\.1 项目执行模板

- **行业化模板**：针对不同合同类型（如软件开发、定制服务等）预设执行流程模板。

    - 示例：软件开发合同模板包含：原型设计、UI 设计、系统设计、前端开发、后端开发、测试、部署运维等标准节点。

- **模板引擎**：模板支持迭代优化，根据历史项目数据自动优化人员排程建议。

#### 3\.5\.2 项目任务与人员管理

- **任务拆解**：将合同执行拆解为具体任务，关联负责人、预计工时。

- **人员能力匹配**：人员按技能、特性、级别进行标签化管理，系统可根据任务类型推荐匹配的执行人员。

- **日报与产物管理**：

    - 执行人员每日提交工作日报，记录工作内容、消耗工时。

    - 支持上传过程产物（需求文档、设计稿、代码包等），自动归档至项目文件库。

    - 示例日报记录：`产品经理 黄xx 进行原型设计 5H，产物上传至项目文件夹\-需求文档`。

#### 3\.5\.3 精细化成本核算

- 基于每日任务的工时与人员成本，自动核算项目的实际人力成本。

- 支持关联项目过程中的报销凭证（出差、招待费等），实现项目全成本的精准核算。

#### 3\.5\.4 人员能力沉淀

- 执行人员每完成一个项目任务，自动积累对应技能的经验值。

- 人员级别随经验自动提升，为后续项目的人员排程提供更精准的数据支撑。

### 3\.6 统一工作台与数据洞察

#### 3\.6\.1 线索 / 项目详情页

将所有业务数据聚合在统一详情页，按业务域分模块展示，实现一站式查看：

1. **销售业务域**：展示线索从新建到跟进、到合同签订的全销售过程数据，包含跟进日志、报价单、报销凭证等。

2. **执行业务域**：

    - **资金子域**：展示回款计划、实际回款、进度跟踪、发票记录。

    - **交付子域**：展示项目流程节点、任务进度、产物文件、团队日报。

#### 3\.6\.2 数据报表与看板

- **销售漏斗分析**：统计线索从公海到签单的各阶段转化率，定位转化瓶颈。

- **项目进度看板**：实时查看所有在执行项目的进度状态、延期风险。

- **销售业绩报表**：按人员 / 部门统计销售额、回款率、跟进效率。

- **成本利润分析**：分析每个项目的投入产出比，优化定价与成本控制。

## 四、业务价值总结

1. **销售提效**：通过公海池流转、智能跟进提醒，解决线索跟进不及时问题，提升线索转化率。

2. **过程透明**：实现从销售到交付的全流程数据留痕，管理层可实时掌握项目进展，避免信息黑箱。

3. **成本精细**：通过分子化的任务与工时管理，实现项目成本的精准核算，提升项目利润率。

4. **能力沉淀**：通过项目数据与人员能力的沉淀，持续优化项目执行模板，实现组织能力的复用与提升。
