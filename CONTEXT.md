# 项目术语表

## 交付计划

基于软件外包项目交付执行 SOP，通过合同识别自动生成的标准化任务管理体系。每个项目可有一份交付计划，包含板块、步骤和里程碑三层结构。

合同是可选的：有合同的项目自动读取交付类型和里程碑；无合同的内部项目在配置面板中手动选择交付类型，不生成里程碑。

### 交付类型 (DeliveryType)

合同的交付物形态分类，决定 SOP 板块四（资质备案 & 上架）的适用步骤。与产品类别（productCategory）是两个独立维度。

- `网站`：触发 ICP 备案、SSL 证书
- `小程序`：触发小程序备案、专项资质、提审
- `APP`：触发 APP 上架准备、提审
- `网站+小程序`：取网站和小程序步骤的并集
- `网站+APP`：取网站和 APP 步骤的并集
- `小程序+APP`：取小程序和 APP 步骤的并集
- `全平台`：取所有步骤的并集

交付类型的来源：
- 有合同：在合同新建/编辑时填写，生成交付计划时只读取
- 无合同（内部项目）：在交付计划配置面板中必填选择

### 板块 (SopPhase)

交付计划的一级结构，对应 SOP 七大板块之一。每个板块有主管（manager），板块状态由其下属步骤自动推导。

| 编号 | 板块名称 | 默认主管映射 |
|------|---------|-------------|
| 1 | 合同交接 | productUsers[0] |
| 2 | 项目启动准备 | owner |
| 3 | 项目交付执行 | owner |
| 4 | 资质备案 & 上架 | opsUsers[0] |
| 5 | 测试验收 | testUsers[0] |
| 6 | 运维支持 | opsUsers[0] |
| 7 | 项目总结 | owner |

板块状态自动推导规则：
- 状态 = 所有步骤中最"落后"的那个
- 起止日期 = 最早步骤开始 ~ 最晚步骤截止
- 不独立编辑，由步骤自动驱动

### 步骤 (SopStep)

交付计划的二级结构，对应 SOP 中的一个具体步骤。步骤编号格式为 `1.1`、`3.7`、`4.5`，自定义步骤格式为 `3-C1`、`3-C2`（C = Custom）。

步骤状态流转规则：
- `pending → in_progress → completed`：严格线性推进
- `completed → in_progress`：允许回退（返工场景）
- 任何非 completed 状态可标记为 `skipped`，跳过后不可恢复

步骤排期基于项目开始日期自动推算，遵循 SOP 依赖模板中的前后置关系，生成后用户可逐条编辑覆盖。

### 里程碑 (SopMilestone)

来自合同 projectMilestones 的关键节点标记。独立于步骤存在，在甘特图上以钻石符号展示。不与 SOP 步骤强制绑定。无合同项目不生成里程碑。里程碑按日期插入对应板块区域展示。

### SOP 依赖模板

定义 SOP 步骤间的前后置关系，用于自动推算排期。关键规则：
- 板块一、二：内部串行
- 板块三：3.1→3.2→3.3 串行，3.4 依赖 3.3，3.5/3.6 与 3.4 可并行，3.7 依赖 3.4，3.8 为贯穿型任务
- 板块四：4.1→4.2 串行，4.3 依赖 4.1，4.4/4.5 依赖 4.3，4.6 依赖 4.4 或 4.5，4.7 独立
- 板块三四并行执行，其余板块串行

### 增量追加

已生成交付计划后再次生成的处理策略。新增板块追加到现有计划，已取消的板块标记跳过，已有步骤保持不变。已完成的步骤进度不会丢失。

### 交付进度环形图

项目详情页统计卡片区的新增卡片，按七大板块分七段展示完成率。点击跳转到交付计划页面。

完成率算法：
- 跳过步骤不计入分母
- 进行中步骤按 50% 折算
- 板块完成率 = (已完成×1 + 进行中×0.5) / (总步骤 - 跳过步骤)
- 整体完成率 = 各板块完成率的简单平均值

### 角色映射

SOP 步骤执行人从角色到项目成员的自动映射规则：

| SOP 角色 | 映射到 |
|----------|--------|
| 产品经理 | productUsers[0] |
| 销售 | salesUsers[0] |
| UI 设计师 | uiUsers[0] |
| 开发主管 | frontendUsers[0] 或 backendUsers[0] |
| 项目经理 | owner |
| 运维专员 | opsUsers[0] |
| 法务专员 | legalUsers[0] |
| 测试工程师 | testUsers[0] |

映射不到的保留角色名，用户可编辑修改。

### 逾期步骤

截止日期 < 今天且状态为 pending 或 in_progress 的步骤。甘特图上横条变红 + 右端警告图标标识。底部汇总条中逾期数标红。
